# 项目说明

## 一、项目总定义

这个项目由两个板块组成：

### 板块一：生成
目标是训练一个模型，根据文本描述生成一个短的、可用于音乐制作的音频 sample。

输入：
- text

输出：
- 一个短音频 sample

### 板块二：移调
目标是把已有 sample 调整到其他音高，同时尽量保持可用性和自然度。

输入：
- 一个已有 sample
- 目标音高

输出：
- 移调后的 sample

这两个板块组合起来，形成一个完整的基本工作流：

1. 先生成一个短 sample  
2. 再把它移调到不同音高使用

---

## 二、当前开发重点

当前优先开发的是：

**板块一：生成模块**

即：

**text -> 短音频 sample**

第二个板块“移调”暂时只做规划，不作为当前主任务。

---

## 三、项目目标

这个项目是为了实习准备，核心目标不是做一个花哨产品，而是做一个能体现机器学习和算法能力的项目。

需要体现的能力包括：

- 基于开源模型进行微调
- 数据构建与清洗
- 条件控制建模
- loss 设计
- 评测设计
- 消融实验

---

## 四、第一板块的版本划分

第一板块分为两个版本：

### v1：无 key 输入
输入只有文本。

形式：
- text -> sample

目标：
- 让模型学会根据文本生成短音频 sample

### v2：有 key 输入
输入为文本加 key。

形式：
- text + key -> sample

目标：
- 在 v1 基础上增加音高控制能力

注意：

- v1 和 v2 的区别只在于**是否输入 key**
- 这和其他概念无关
- 当前不考虑 reference audio
- 当前不做其他额外控制变量

---

## 五、项目不做什么

为了保证项目范围可控，当前版本不做以下内容：

- 不做整首音乐生成
- 不做旋律草稿生成
- 不做 reference audio 条件输入
- 不做插件实现
- 不做完整虚拟乐器
- 不做 velocity layer
- 不做 articulation
- 不做循环点优化
- 不做多采样全键盘映射
- 不做长音 sustain 设计
- 不做复杂 UI

---

## 六、为什么拆成“生成 + 移调”两个板块

因为这两个问题本质不同。

### 生成
是在“创造一个新的声音”。

### 移调
是在“保持已有声音的大体身份，只改变音高”。

前者是生成问题。  
后者更接近变换问题。

如果把这两件事硬塞进一个模型，目标会混乱。  
拆开之后，问题定义更清楚，系统结构也更合理。

---

## 七、第一板块的输出范围

为了让项目可做，第一板块的输出必须严格收缩。

建议第一版只做：

- 时长约 1–2 秒
- 短音频 sample
- 尽量优先选择主音高相对稳定的素材

第一版不要故意去做：

- 明显滑音
- 短旋律
- 内部音高变化很强的片段
- 长 loop
- 长时间 evolving sound

原因是后续第二板块要做移调，如果第一板块本身输出的 sample 音高不稳定，后续会很难处理。

---

## 八、数据形式

第一板块当前最小数据形式：

### v1
- audio
- text

### v2
- audio
- text
- key

当前不需要单独的 category 字段，除非后续评测阶段证明它确实有必要。

---

## 九、建模方向

这个项目不从零训练基础模型。  
应选择一个开源 text-to-audio 或 text-to-music 模型作为底座，然后在目标任务上做微调。

基本路线：

1. 选择底座模型  
2. 构建和清洗数据集  
3. 先训练 v1 baseline  
4. 再训练 v2  
5. 设计并加入 key 相关 loss  
6. 做评测与消融实验

---

## 十、loss 设计原则

### v1
v1 不涉及 key，因此只需要原模型的生成 loss。

形式：
- L = L_gen

### v2
v2 加入 key 输入后，可以加入一个额外的 key loss。

基本形式：
- L = L_gen + λL_key

其中：

- L_gen：原始生成 loss
- L_key：输出 sample 的音高和目标 key 的匹配损失

如果用于提取音高的函数不可微，则不要直接把它放进训练图里。

这时可选做法是：

1. 只把不可微函数用于离线评测  
2. 训练一个可微的 key classifier 或 pitch regressor 作为代理 head  
3. 用这个可微 head 的输出定义 L_key

如果采用分类做法，可以让代理 head 输出离散类别。  
如果采用回归做法，可以输出离散 MIDI note 对应的预测值。  
第一版更推荐做**离散分类**，因为更稳。

---

## 十一、评测原则

项目不能只靠主观听感。  
至少要做以下评测：

### v1
- 生成质量
- 文本对齐程度

### v2
- 生成质量
- 文本对齐程度
- key 控制效果

可重点比较：

- v1：text-only
- v2：text + key
- v2 + key loss

---

## 十二、为什么这个项目适合面试

这个项目的价值不在于“做了一个 AI 音乐工具”。

真正能拿来面试讲的是：

- 你如何把问题拆成两个板块
- 你为什么先做生成，再做移调
- 你为什么把 v1 / v2 定义成无 key / 有 key
- 你如何从 baseline 逐步加控制项
- 你如何处理不可微音高函数
- 你如何做评测和消融

这些内容能清楚展示你的机器学习建模能力。

---

## 十三、一句话总结

这是一个由**生成 + 移调**两块组成的音乐制作项目。  
当前主任务是先完成第一块，并把它分成：

- v1：text -> sample
- v2：text + key -> sample
